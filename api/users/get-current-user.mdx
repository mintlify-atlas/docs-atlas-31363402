## Get Current User

Retrieve information about the currently authenticated user.

### Endpoint

```
GET /api/me
```

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Response

<ResponseField name="user" type="object">
  User information
  
  <Expandable title="user object">
    <ResponseField name="id" type="string">
      User UUID
    </ResponseField>
    
    <ResponseField name="username" type="string">
      Username
    </ResponseField>
    
    <ResponseField name="role" type="string">
      User role: `admin` or `user`
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of account creation
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

<CodeGroup>

```bash curl
curl https://your-umami-instance.com/api/me \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://your-umami-instance.com/api/me', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
  },
});

const user = await response.json();
console.log(`Logged in as: ${user.user.username}`);
```

```python Python
import requests

response = requests.get(
    'https://your-umami-instance.com/api/me',
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)

user = response.json()
print(f"Logged in as: {user['user']['username']}")
```

</CodeGroup>

### User Roles

<ResponseField name="admin">
  Full system access including:
  - Manage all websites
  - Manage all teams
  - Manage all users
  - Access all analytics data
  - System configuration
</ResponseField>

<ResponseField name="user">
  Standard user access:
  - Manage own websites
  - Create and join teams
  - View team resources based on team role
  - Cannot manage other users
</ResponseField>

### Use Cases

- Verify authentication status
- Display user information in UI
- Check user permissions
- Validate token before making other API calls

### Error Responses

<ResponseField name="401" type="error">
  Unauthorized - Invalid or expired token
</ResponseField>

<Note>
  This endpoint returns the user information associated with the provided Bearer token. Use this to verify authentication and retrieve the current user's details.
</Note>
